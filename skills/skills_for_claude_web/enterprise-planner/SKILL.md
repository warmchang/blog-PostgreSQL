# Enterprise Planner — 企业未来规划与建议书

## 总体流程

```
1. 信息采集 → 2. 企业画像构建 → 3. 外部环境分析 → 4. 战略规划撰写 → 5. 输出报告
```

---

## Step 1：信息采集

### 1.1 从用户输入中提取

用户通常会提供（至少其中之一）：
- 企业名称
- 企业简介 / 一句话描述
- 公司网站 URL
- 所在行业 / 主营业务

### 1.2 主动网络搜索

使用 `web_search` 和 `web_fetch` 工具，围绕以下维度并行搜索：

```
搜索顺序（优先级由高到低）：

A. 企业基本面
   - "[企业名] 营收 利润 2023 2024"
   - "[企业名] 商业模式 产品线"
   - "[企业名] 融资 上市 投资人"
   - "[企业名] 新闻 最新动态"

B. 行业与市场
   - "[所在行业] 市场规模 2024 2025"
   - "[所在行业] 行业报告 趋势"
   - "[所在行业] 竞争格局 龙头企业"

C. 竞争对手
   - "[企业名] 竞争对手 对比"
   - "[行业] 头部企业 市场份额"

D. 宏观政策
   - "[行业] 政策 监管 2024"
   - "[行业] AI 数字化 机会"
```

**如用户提供了网站 URL，必须使用 `web_fetch` 抓取完整内容。**

## Step 2：构建企业画像

将采集到的信息整理为结构化画像，涵盖内部与外部、当下与未来四个象限：

### 内部 × 当下 — 业务体质诊断

| 维度 | 分析要点 |
|------|---------|
| 财务健康 | 营收规模、增速、利润率、现金流状况（若无数据，注明"信息有限，推测...") |
| 产品/服务结构 | 各业务线的收入贡献、增速、竞争力 |
| 客户资产 | 客户类型、集中度、付费逻辑、核心痛点 |
| 核心能力 | 驱动模式（技术/营销/运营效率）、护城河深度 |

### 内部 × 未来 — 战略意图与进化力

- 创始人/核心团队背景与野心判断
- 过去1-3年最大的结构性变化
- 现有能力可迁移的新赛道

### 外部 × 当下 — 竞争与市场温度

- 市场利润池分布与价值链位置
- 直接竞争对手对比（优/劣势差异化）
- 替代品威胁与潜在进入者

### 外部 × 未来 — 结构性机会与风险

- 技术非连续性（AI、自动化等颠覆点）
- 人口与社会结构变迁
- 政策长期锚点（双碳、数字经济等）
- 消费价值观迁移

## Step 3：战略规划撰写

报告必须包含以下章节（顺序可根据企业情况调整）：

### 必须包含的章节

1. **执行摘要** — 3-5 句话总结核心建议，让读者5秒内抓住精髓

2. **企业现状诊断**
   - 核心优势与短板（SWOT 或自定义框架）
   - 当前所处的竞争位置

3. **行业与赛道分析**
   - 市场规模与增速（附数据来源）
   - 价值链结构与利润池
   - 竞争格局地图（用 mermaid 或 SVG 可视化）

4. **机会识别与场景拆解**
   - 至少 2-3 个具体场景/赛道机会
   - 每个机会说明：为什么现在、为什么这家企业、如何切入

5. **商机发掘与痛点分析**
   - 客户真正愿意付费的痛点（区分"伪需求"与"真刚需"）
   - 商业模式设计（如何变现）

6. **产品/服务原型设计**
   - MVP（最小可行产品）定义
   - 产品路线图（用 mermaid 时间线展示）
   - 核心功能优先级

7. **团队与资源建议**
   - 关键岗位配置建议
   - 需要引入的核心能力缺口

8. **战略路线图**
   - 0-6个月：生存期目标
   - 6-18个月：成长期目标
   - 18-36个月：扩张期目标
   - 用 mermaid gantt 或时间线可视化

9. **风险与前提假设**
   - 明确列出规划成立的核心假设
   - 每个假设崩塌时的备选方案

10. **行动优先级清单**
    - Top 5 优先级行动（按 ROI 排序）

## Step 4：内容质量标准

### 数据与证据要求
- ✅ 引用来源明确（市场研究机构、行业协会、上市公司财报等）
- ✅ 使用区间数据而非精确数字（"市场规模约 800-1200 亿元"）
- ✅ 每个核心论点至少有 1 个支撑案例
- ❌ 避免用个例以偏概全
- ❌ 避免无来源的"据悉"、"业内人士透露"

### 逻辑结构要求
- 每个战略建议必须有明确的前提假设
- 假设崩塌时的备选路径必须写出来
- 论点→论据→结论 三段式结构

### 图表使用规范

**Mermaid 图表（优先使用）：**
```
- 竞争格局：quadrantChart（四象限定位图）
- 产品路线图：gantt
- 业务流程：flowchart
- 组织架构：graph TD
- 时间轴：timeline
```

**⚠️ Mermaid 中文渲染规则（必须严格遵守）：**

1. **quadrantChart 象限图** — 所有中文标签必须用双引号包围：
```
quadrantChart
    title "竞争格局定位"
    x-axis "低市场份额" --> "高市场份额"
    y-axis "低增长" --> "高增长"
    quadrant-1 "明星"
    quadrant-2 "问题"
    quadrant-3 "瘦狗"
    quadrant-4 "现金牛"
    "企业A": [0.7, 0.8]
    "竞品B": [0.4, 0.5]
```

2. **gantt 时间线** — 中文任务名必须用双引号包围；**日期格式只能用 `YYYY-MM-DD` 或 `MM-DD`，禁止使用 Q1/Q2/Q3/Q4 等季度格式**：
```
gantt
    title "产品路线图"
    dateFormat MM-DD
    section "第一阶段"
        "需求调研"    :done, 01-01, 02-28
        "原型设计"    :active, 03-01, 04-30
    section "第二阶段"
        "开发上线"    : 05-01, 08-31
```

3. **timeline 时间轴** — 中文文本必须用双引号包围，**section 后的标题也需双引号**：
```
timeline
    title "战略里程碑"
    section "2025"
        "01月" : "完成MVP"
        "06月" : "首批客户"
    section "2026"
        "03月" : "规模扩张"
```

4. **flowchart / graph** — 节点标签中含中文时必须用双引号或方括号包围：
```
flowchart TD
    A["战略分析"] --> B["机会识别"]
    B --> C["产品设计"]
```

**SVG 图表（复杂可视化用）：**
- 市场份额饼图
- 价值链流转图
- 雷达图（能力评估）

**Text 表格：**
- 竞品对比矩阵
- 优先级排序

## Step 5：输出规范

### 文件保存
```bash
# 保存到 markdown/ 目录
文件名格式：[企业名]-未来规划建议书-[YYYY-MM].md
示例：字节跳动-未来规划建议书-2025-01.md
```

### Markdown 格式要求
- 使用 H1-H4 标题层级，H1 只用一次（报告标题）
- 重要结论用 `> ` 引用块突出
- 关键数字**加粗**
- 报告末尾附"参考资料"章节，列出搜索到的核心来源

### 报告长度
- 最短：4000 字（含图表）
- 理想：6000-10000 字
- 上限：视复杂度而定，不强制压缩

---

## 特殊情况处理

### 信息严重不足时
若搜索后企业信息仍极为有限（如极早期创业公司），应：
1. 告知用户信息来源局限
2. 基于行业通用规律进行推演，并明确标注"基于行业推演"
3. 重点输出行业分析和商业模式框架，而非企业个性化诊断

### 涉及竞品敏感信息时
- 只引用公开资料（官网、公开报道、上市公司公告）
- 不捏造竞品数据

### 输出语言
- 默认与用户使用的语言一致
- 若用户输入为中文，报告为中文；英文同理
