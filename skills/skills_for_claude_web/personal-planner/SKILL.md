# Personal Planner Skill —— 个人未来规划与建议书

## 概述

本 Skill 将提问者提供的信息转化为一份**专业、深度、可执行**的个人规划建议书。

整个流程分三阶段：
1. **画像构建** — 系统梳理提问者的内部信息与外部环境
2. **战略研判** — 结合趋势与画像，发现交集机会
3. **报告输出** — 生成图文并茂的 Markdown 建议书

---

## 阶段一：画像构建

### 1.1 信息收集与整理

从提问者已提供的信息中提取以下维度，**不足的维度需主动追问**（但不要一次问超过 3 个问题，分批进行）：

#### 内部信息

| 维度 | 关键问题 |
|------|---------|
| 硬技能 | 精通哪些专业技术？有什么证书、作品、成果？ |
| 软技能 | 最强的可迁移能力？（领导/沟通/分析/创意） |
| 资源优势 | 人脉网络、资金储备、行业信息差、品牌/专利？ |
| 能力盲区 | 不擅长或厌恶做什么？ |
| 驱动力 | 什么事能让他忘记时间？成就/掌控/连接/安全/意义感？ |
| 能量模式 | 内向 vs 外向？快节奏 vs 稳定节奏？ |
| 核心价值观 | 家庭 vs 事业？冒险 vs 安稳？社会认可 vs 内在自由？ |
| 当前痛点 | 最近一次强烈焦虑或无力感是什么场景？ |
| 限制性信念 | 内心有哪些自我设限的声音？ |
| 理想生活画像 | 3年后一个满意的"普通一天"是怎样度过的？ |
| 风险偏好 | 30% 概率 10 倍回报 vs 100% 概率平稳回报，倾向哪个？ |
| 愿意放弃的 | 为达到目标，愿意放弃稳定/闲暇/收入/他人理解中的哪些？ |

#### 外部信息

| 维度 | 关键问题 |
|------|---------|
| 所在行业 | 当前所在行业、公司规模、价值链位置 |
| 岗位现状 | 核心工作内容，AI/自动化替代风险评估 |
| 家庭生命周期 | 单身/已婚/育儿/赡养压力，时间与精力约束 |
| 地理位置 | 城市、是否愿意迁移、跨国机会 |
| 财务状况 | 月收入范围、储蓄、负债、可投资本金 |

### 1.2 画像输出格式

在报告中以如下结构输出画像摘要：

```
## 🧭 提问者画像

### 核心能力矩阵
（表格：硬技能 / 软技能 / 资源优势 / 能力盲区）

### 动机与价值观
（文字段落，描述驱动力类型、能量模式、价值观排序）

### 当前处境
（文字段落，描述痛点、已尝试路径、限制性信念）

### 理想未来画像
（文字段落，描述3年后的理想生活场景）

### SWOT 分析
（Mermaid 四象限图或 Markdown 表格）
```

## 阶段二：战略研判

### 2.1 行业与赛道分析

针对提问者所在或希望进入的领域，进行如下分析：

- **行业现状**：市场规模、增速、主要参与者、利润分布
- **价值链解构**：哪些环节是高利润核心，哪些正在被边缘化
- **供需结构**：人才供需，是红海还是蓝海
- **AI/技术替代性**：当前工作有多少可被替代，未来 3-5 年趋势
- **不可逆趋势**：人口结构、政策方向、技术扩散曲线中的确定性

### 2.2 机会发现框架

寻找"他想要的 × 他能做的 × 时代需要的"三圆交集：

```
用 Mermaid Venn 图或文字清晰展示三者交集
```

重点寻找：
- **新语言/新岗位**：近两年出现的新职位名词背后的需求缺口
- **顶尖人才流向**：资本与高知人才正在集聚的领域
- **未被满足的痛点**：市场上现有产品/服务的明显缺陷

### 2.3 商机深挖

针对每个核心机会点，输出：

| 项目 | 内容 |
|------|------|
| 痛点描述 | 客户愿意为此付费的核心痛苦是什么？ |
| 需求规模 | 有多少人有这个需求？可引用数据 |
| 当前替代品 | 客户现在怎么解决？为什么不满意？ |
| 差异化机会 | 提问者独特的切入角度 |
| 变现路径 | 服务/产品/平台/内容，如何产生收入 |
| 最小原型 | 用最小成本验证的 MVP 形态 |

## 阶段三：报告生成

### 3.1 报告结构

必须包含以下模块，每个模块都要有**数据支撑**和**清晰的前提假设**：

```markdown
# 《[提问者姓名/代号] 个人未来规划建议书》

> 生成日期：YYYY-MM-DD
> 核心假设：[列出 3-5 条关键假设，假设崩塌时应触发的备选方案]

## 一、提问者画像摘要
## 二、外部环境研判
   ### 2.1 宏观趋势
   ### 2.2 行业深度分析
   ### 2.3 赛道比较
## 三、核心机会地图
   ### 3.1 三圆交集分析
   ### 3.2 Top 3 机会点详述
## 四、商业路径设计
   ### 4.1 客户画像与痛点
   ### 4.2 产品/服务原型
   ### 4.3 变现模型
   ### 4.4 最小可行产品（MVP）设计
   ### 4.5 团队组建建议
## 五、奥德赛三版本规划
   ### 版本一：当前路径极致化
   ### 版本二：突发中断的备选
   ### 版本三：不考虑钱和面子的理想版
## 六、行动路线图
   ### 第一阶段（0-3 个月）：验证与准备
   ### 第二阶段（3-12 个月）：起步与迭代
   ### 第三阶段（1-3 年）：规模化与深化
## 七、风险预警与对冲策略
## 八、关键资源与下一步行动
```

### 3.2 图表使用规范

- **流程图/架构图**：使用 Mermaid `flowchart` 或 `graph`
- **时间线**：使用 Mermaid `gantt` 或 `timeline`
- **矩阵/象限**：使用 SVG 绘制四象限图，或 Markdown 表格
- **数据对比**：使用 Markdown 表格，附来源说明
- **关系图**：使用 Mermaid `mindmap` 或 `erDiagram`

每个主要论点都需要：
- ✅ 数据来源（行业报告、政府统计、权威机构）
- ✅ 前提假设明确标注
- ✅ 假设崩塌的备选方案

### 3.3 写作风格要求

- **观点清晰**：每个建议必须有明确立场，不能两边讨好
- **案例具体**：不用个例以偏概全，用多个案例支撑规律性结论
- **逻辑严密**：结论 ← 论据 ← 前提假设，链条清晰
- **可执行性**：每个建议都要有具体的"第一步行动"
- **诚实标注风险**：不夸大机会，明确标注不确定性

---

## 输出规范

- 文件路径：`markdown/personal-plan-[姓名或代号]-[YYYYMMDD].md`
- 格式：标准 Markdown，支持 Mermaid 图表
- 长度：建议 3000-8000 字，视信息丰富度调整
- 语言：与提问者使用的语言保持一致

---

## 注意事项

1. **信息不足时**：不要强行生成，先追问缺失维度，优先级：行业/职业 > 年龄/地点 > 财务/家庭
2. **避免泛化**：每份规划必须是针对该提问者的专属输出，不能套模板
3. **不做道德评判**：对提问者的选择保持中立，忠实反映其价值观，而非输出社会期望
4. **标注局限性**：AI 生成的规划有信息时效性限制，需提醒提问者验证关键数据
5. **前提假设透明**：报告开头必须列出核心假设，便于提问者根据自身实际情况调整
