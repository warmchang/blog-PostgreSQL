## 背景    
### 11.3.3 实时分析与连续聚合  
  
在时序数据库应用中，实时分析（Real-time Analytics） 是最核心的能力之一。例如：  
  
* 运维监控系统需要实时查看 CPU 过去5分钟平均值  
* IoT 平台需要统计 每小时设备数据均值  
* 金融交易系统需要计算 分钟级K线  
  
如果直接对原始数据进行聚合计算，随着数据规模增长，查询成本会迅速增加。例如：  
  
```sql  
SELECT  
    time_bucket('1 hour', time) AS hour,  
    avg(cpu_usage)  
FROM metrics  
GROUP BY hour;  
```  
  
当数据量达到 数十亿行 时，这类查询将变得非常昂贵。  
  
为了解决这一问题，TimescaleDB 提出了 Continuous Aggregates（连续聚合） 技术，使数据库能够在后台自动维护聚合结果，从而实现高效的实时分析。  
  
   
  
#### 一、连续聚合的设计思想  
  
连续聚合本质上是 自动维护的物化视图（Materialized View）。  
  
在传统 PostgreSQL 中，物化视图需要手动刷新：  
  
```sql  
REFRESH MATERIALIZED VIEW view_name;  
```  
  
这种方式存在明显问题：  
  
* 刷新期间无法使用视图  
* 刷新开销巨大  
* 无法实时更新  
  
TimescaleDB 的连续聚合解决了这些问题。  
  
其核心思想是：  
  
```text  
原始数据  
   ↓  
增量计算  
   ↓  
聚合结果表  
```  
  
数据库在后台自动维护聚合结果，仅对新增数据进行计算，从而显著降低计算成本。  
  
   
  
#### 二、时间分桶（Time Bucket）  
  
连续聚合的基础是 时间分桶（Time Bucket）。  
  
在时序分析中，数据通常按照固定时间窗口进行聚合，例如：  
  
* 每分钟  
* 每小时  
* 每天  
  
TimescaleDB 提供了 `time_bucket()` 函数实现这一功能。  
  
示例：  
  
```sql  
SELECT  
    time_bucket('1 hour', time) AS hour,  
    avg(cpu_usage)  
FROM cpu_metrics  
GROUP BY hour;  
```  
  
`time_bucket()` 会将时间映射到固定时间窗口，例如：  
  
| time  | bucket |  
| ----- | ------ |  
| 10:05 | 10:00  |  
| 10:20 | 10:00  |  
| 10:59 | 10:00  |  
  
这种设计使数据库能够高效执行时间窗口聚合。  
  
   
  
#### 三、创建连续聚合  
  
TimescaleDB 使用 SQL 语法创建连续聚合。  
  
示例：  
  
```sql  
CREATE MATERIALIZED VIEW cpu_hourly  
WITH (timescaledb.continuous)  
AS  
SELECT  
    time_bucket('1 hour', time) AS bucket,  
    host,  
    avg(cpu_usage) AS avg_cpu  
FROM cpu_metrics  
GROUP BY bucket, host;  
```  
  
该语句创建了一个 连续聚合视图。  
  
其特点包括：  
  
* 数据自动刷新  
* 查询无需扫描原始表  
* 支持实时更新  
  
数据库会在后台自动计算新的聚合结果。  
  
   
  
#### 四、连续聚合的刷新机制  
  
连续聚合采用 增量刷新（Incremental Refresh）。  
  
工作流程如下：  
  
```text  
新数据写入  
     ↓  
识别受影响时间窗口  
     ↓  
重新计算该窗口  
     ↓  
更新聚合结果  
```  
  
例如：  
  
假设统计 每小时 CPU 平均值：  
  
```text  
10:00-11:00  
```  
  
当新的数据写入该时间窗口时，数据库只需要重新计算这一小时的数据，而不必扫描整个表。  
  
根据 TimescaleDB 官方测试，在 10亿行数据情况下，连续聚合查询性能相比实时聚合可提升 100倍以上。  
  
   
  
#### 五、自动刷新策略  
  
连续聚合通常需要定义自动刷新策略。  
  
例如：  
  
```sql  
SELECT add_continuous_aggregate_policy(  
    'cpu_hourly',  
    start_offset => INTERVAL '1 day',  
    end_offset => INTERVAL '1 hour',  
    schedule_interval => INTERVAL '5 minutes'  
);  
```  
  
参数含义：  
  
| 参数                | 含义     |  
| ----------------- | ------ |  
| start_offset      | 计算范围起点 |  
| end_offset        | 计算范围终点 |  
| schedule_interval | 刷新周期   |  
  
示例逻辑：  
  
```text  
每5分钟刷新  
过去1天的数据  
但忽略最近1小时  
```  
  
这样可以避免频繁刷新正在写入的数据。  
  
   
  
#### 六、实时查询（Real-time Aggregation）  
  
TimescaleDB 还支持 实时查询模式。  
  
其核心机制：  
  
```text  
已聚合数据 + 最新原始数据  
```  
  
查询流程：  
  
1. 从连续聚合表读取历史结果  
2. 从原始表读取最新数据  
3. 自动合并结果  
  
示例查询：  
  
```sql  
SELECT *  
FROM cpu_hourly  
ORDER BY bucket DESC  
LIMIT 10;  
```  
  
数据库会自动返回最新数据。  
  
这种设计保证：  
  
* 查询速度快  
* 数据始终接近实时  
  
   
  
#### 七、多层级聚合（Hierarchical Aggregation）  
  
在大型系统中，通常需要多个时间粒度的聚合。  
  
例如：  
  
```text  
秒级数据  
 ↓  
分钟聚合  
 ↓  
小时聚合  
 ↓  
天聚合  
```  
  
TimescaleDB 支持 层级连续聚合。  
  
示例：  
  
```sql  
CREATE MATERIALIZED VIEW cpu_daily  
WITH (timescaledb.continuous)  
AS  
SELECT  
    time_bucket('1 day', bucket),  
    host,  
    avg(avg_cpu)  
FROM cpu_hourly  
GROUP BY 1,2;  
```  
  
这种方式可以形成 多级分析体系。  
  
优势：  
  
* 减少计算量  
* 提高查询性能  
  
   
  
#### 八、企业应用案例  
  
连续聚合广泛应用于多个行业。  
  
   
##### 1 运维监控平台  
  
监控系统通常需要统计：  
  
* 1分钟 CPU平均值  
* 5分钟负载  
* 1小时资源使用率  
  
例如监控平台 Prometheus 中的 recording rules 与 TimescaleDB 连续聚合设计非常类似。  
  
   
  
##### 2 物联网平台  
  
IoT 数据通常需要多层级分析：  
  
```text  
原始数据 → 每分钟统计 → 每小时统计 → 每日统计  
```  
  
连续聚合可以显著减少实时查询开销。  
  
   
  
##### 3 金融行情分析  
  
金融系统常见的分析包括：  
  
* 分钟K线  
* 小时K线  
* 日K线  
  
例如：  
  
| 时间 | 开盘 | 收盘 | 成交量 |  
| -- | -- | -- | --- |  
  
这些统计可以通过连续聚合自动维护。  
  
   
  
#### 九、性能优势  
  
连续聚合的性能优势主要体现在：  
  
| 方式   | 查询复杂度    |  
| ---- | -------- |  
| 实时聚合 | O(N)     |  
| 连续聚合 | O(聚合窗口数) |  
  
例如：  
  
```text  
10亿条原始数据  
→ 1万条小时聚合数据  
```  
  
查询速度可以提升 1000倍以上。  
  
   
  
#### 小结  
  
连续聚合是 TimescaleDB 的核心分析能力之一。  
  
其关键技术包括：  
  
* 时间分桶（Time Bucket）  
* 增量刷新  
* 实时聚合  
* 层级聚合  
  
通过这些技术，TimescaleDB 可以在海量时序数据上实现高效实时分析，同时保持 PostgreSQL 的 SQL 兼容性。
